Design System

This document serves as the single source of truth for all components, tokens, and patterns used across the portfolio. Every UI element should reference this system to maintain visual and functional consistency.

How to Use This System

  1. Colors: Use CSS custom properties from style.css. Never hardcode hex values.
  2. Spacing: Use var(--pad) for horizontal padding, var(--max-w) for max width.
  3. Typography: Use var(--font) for body, 'Chakra Petch' for headings, 'IBM Plex Mono' for code/specs.
  4. Buttons: Use .btn + modifier class. For pill shape, add border-radius: 999px.
  5. Cards: Use .pcard for large project cards, .fcard for feature cards, .card for generic cards.
  6. Tags: Use .tag class for all category labels.
  7. Animations: Add .fade-in class to elements that should animate on scroll.
  8. Navigation: Keep the .nav structure consistent across all pages.
